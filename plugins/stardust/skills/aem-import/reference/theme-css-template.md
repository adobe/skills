# Per-theme stylesheet template (cascade-layer scaffold)

Every per-theme stylesheet generated by `stardust:aem-import` follows
this scaffold. The layer order is **load-bearing** — it eliminates the
specificity / source-order traps catalogued in
[`conventions.md` §7](./conventions.md#7-per-variant-inheritance-traps)
by making the cascade explicit. Variants always win against base and
substrate, regardless of selector specificity or where they appear in
the file.

## Why cascade layers

Before this scaffold, theme CSS was written as if each variant rule
could stand alone. In practice every variant fought three implicit
layers (browser reset, shared base, substrate override), and any miss
silently created a bug — wrong text color on dark, centered text
left-aligned, image cropped instead of contained.

CSS Cascade Layers (`@layer`) make the cascade explicit:

- Rules in a later layer beat rules in an earlier layer **regardless
  of selector specificity or source order**.
- Within a layer, normal specificity + source-order rules apply.

By assigning `base`, `substrate`, `variant` to ascending layers, the
generator can write naïve variant selectors (e.g.
`.cards.locations .cards-card-body p { color: white }`) and they will
ALWAYS win against the shared base — no specificity bumps, no source
ordering, no `!important`.

## The scaffold

```css
/* =================================================================
 * <theme>.css — per-theme stylesheet, AEM EDS blocks mode
 * Generated by stardust:aem-import.
 *
 * Layer order — later layers ALWAYS beat earlier, regardless of
 * selector specificity. Variants in @layer variant therefore win
 * automatically against base + substrate, eliminating the
 * specificity wars catalogued in conventions.md §7.
 *
 *   tokens     → :root design tokens (palette, type scale, spacing)
 *   fonts      → @font-face declarations
 *   reset      → CSS reset / normalize
 *   chrome     → utility-strip / header / mega-nav / footer
 *   base       → generic block defaults (.hero.block, .text.block,
 *                .cards.block, .columns.block, .cta-bar.block)
 *   layout     → inter-block spacing, section substrates' base
 *   substrate  → per-substrate overrides (.section.dark, .warm-stone)
 *   variant    → per-block variants (.cards.tiers, .cards.logos, ...)
 *   responsive → @media queries (cascade applies normally inside)
 * ================================================================= */

@layer tokens, fonts, reset, chrome, base, layout, substrate, variant, responsive;

/* ---------- tokens ---------- */
@layer tokens {
  :root {
    --color-page: #ffffff;
    --color-body: #454545;
    /* ... all design tokens ... */
  }
}

/* ---------- fonts ---------- */
@layer fonts {
  @font-face { font-family: "Roboto"; src: url("/assets/<theme>/fonts/roboto.woff2") format("woff2"); }
  /* ... */
}

/* ---------- reset ---------- */
@layer reset {
  *, *::before, *::after { box-sizing: border-box; }
  body { margin: 0; font-family: var(--font-body); ... }
  img { max-width: 100%; height: auto; display: block; }
  a { color: inherit; text-decoration: none; }
}

/* ---------- chrome ---------- */
@layer chrome {
  .utility-strip { ... }
  .site-header { ... }
  .mega-nav { ... }
  .site-footer { ... }
}

/* ---------- base ---------- */
@layer base {
  /* Container width control */
  main .section > div { max-width: var(--container-max); margin: 0 auto; padding: 0 var(--sp-lg); }

  /* Generic block defaults */
  .hero.block { display: grid; min-height: 520px; ... }
  .text.block { max-width: 880px; margin: 0 auto; }
  .cards.block > ul { display: grid; gap: var(--sp-lg); list-style: none; padding: 0; }
  .cards.block > ul > li { display: flex; flex-direction: column; gap: var(--sp-md); }
  .cards.block .cards-card-body p:not(.button-wrapper) { color: var(--color-body); }
  .cards.block .cards-card-image img { width: 100%; height: 100%; object-fit: cover; }
  .columns.block > div { display: grid; grid-template-columns: 1fr 1fr; gap: var(--sp-xxl); }
  .cta-bar.block { padding: var(--sp-section) 0; background: var(--color-surface-dark); }

  /* EDS button decoration */
  a.button { display: inline-flex; align-items: center; padding: 12px 24px; ... }
  a.button.primary { background: var(--color-cat-yellow); color: var(--color-text-on-yellow); }

  /* Icon img sizing (when EDS injects <img src=icon.svg>) */
  .icon > img { width: 1em; height: 1em; vertical-align: -0.15em; }
  .icon:has(> img)::before { content: none !important; }
}

/* ---------- layout ---------- */
@layer layout {
  /* Section padding defaults — substrate layer overrides per-substrate */
  main .section { padding: var(--sp-section) 0; }
  main .section.hero-container { padding: 0; }   /* hero is full-bleed */
  main .section:not(:has(.block)) { display: none; }   /* hide empty metadata-only sections */

  /* Inter-block spacing — EDS adds no gap between block wrappers in a section */
  .cards-wrapper + .text-wrapper,
  .columns-wrapper + .text-wrapper,
  .cards-wrapper + .cards-wrapper,
  .columns-wrapper + .cards-wrapper {
    margin-top: var(--sp-xxl);
  }
}

/* ---------- substrate ---------- */
@layer substrate {
  main .section.warm-stone { background: var(--color-warm-stone); }
  main .section.dark { background: var(--color-surface-dark); color: var(--color-text-on-dark); }
  main .section.dark a { color: var(--color-text-on-dark); }

  /* Base block overrides under substrates.
     Note: these no longer trap variants — @layer variant wins regardless. */
  .section.dark .text.block { text-align: left; }
  .section.dark .cards.block .cards-card-body p:not(.button-wrapper) { color: rgba(255,255,255,0.85); }
}

/* ---------- variant ---------- */
@layer variant {
  /* Per-block variants — automatically win against base + substrate. */

  /* Text variants */
  .text.block.centered { text-align: center; max-width: 880px; margin: 0 auto; }

  /* Hero variants */
  .hero.block.service a.button.primary { background: transparent; border: 1px solid var(--color-cat-yellow); }

  /* Cards variants */
  .cards.tiers > ul { grid-template-columns: repeat(4, minmax(0, 1fr)); }
  .cards.rail > ul { grid-template-columns: repeat(5, minmax(0, 1fr)); }
  .cards.photos > ul > li { aspect-ratio: 3/4; position: relative; overflow: hidden; }
  .cards.photos > ul > li {
    /* MUST set justify-content explicitly — flex-direction inherits column from base,
       so align-items: flex-end aligns the cross axis (right), not bottom. */
    justify-content: flex-end;
    align-items: stretch;
  }

  /* Variants holding wide-aspect images default to contain, not cover */
  .cards.logos .cards-card-image img,
  .cards.logos .cards-card-body img {
    object-fit: contain;
    height: 64px;
    max-height: 64px;
  }

  /* Single-cell-row tolerance — cards.js can't classify single-cell rows
     as image cells. Variants that author single-cell rows must handle
     both cards-card-image AND cards-card-body img paths. */
}

/* ---------- responsive ---------- */
@layer responsive {
  @media (max-width: 1024px) { ... }
  @media (max-width: 768px) { ... }
  @media (max-width: 640px) { ... }
}
```

## Layer order rationale

| Layer | Why this position |
|---|---|
| **tokens** | First so all later layers can use `var(--...)` |
| **fonts** | Before reset so `body { font-family }` resolves the family |
| **reset** | Browser defaults override; everything after assumes a clean slate |
| **chrome** | Owns its own scope (`.utility-strip`, `.site-header`); never participates in block specificity wars but loads early so chrome paints during EDS's main-content render |
| **base** | Block defaults — variants will override these |
| **layout** | Inter-section + inter-block defaults; variant-agnostic |
| **substrate** | Per-substrate overrides on base blocks — variants STILL win because they're in a later layer |
| **variant** | Per-variant rules — always win against base + substrate without specificity tricks |
| **responsive** | Last so media queries can adjust anything; normal specificity applies inside |

## Browser support

CSS Cascade Layers ship in **all evergreen browsers since 2022** (Chrome
99, Firefox 97, Safari 15.4). EDS targets the same baseline. Older
browsers ignore `@layer` blocks entirely — the rules inside the blocks
are parsed but live in the "unlayered" cascade, where their normal
specificity applies. In practice that means the theme degrades to the
pre-cascade-layer behavior (with all the §7 traps re-enabled), which
is no worse than before the refactor.

No transpilation, no PostCSS plugin, no polyfill needed.

## Migration of existing themes

Existing pre-layer themes continue to work — they're untouched. If a
theme needs refactoring (e.g., to add new variants without hitting §7
traps), wrap the existing content in `@layer base { ... }` /
`@layer substrate { ... }` blocks and the cascade behavior shifts
automatically. No selector rewrites needed.

When the skill regenerates a theme that was authored pre-layer, it
re-emits using the scaffold. The CSS output volume grows by ~10 lines
of `@layer` declarations and gains the layer wrappers — no semantic
diff beyond layer assignment.

## What the generator emits per layer

The skill walks Phase 2 with explicit layer-by-layer assignment:

1. **tokens** — translated from the prototype's `:root` block
2. **fonts** — `@font-face` declarations, asset paths rewritten
3. **reset** — the standard reset block (template constant)
4. **chrome** — translated from the prototype's chrome CSS
5. **base** — generic block selectors derived from Phase 1 mapping +
   the standard EDS-pipeline defensive defaults from
   `conventions.md` §4
6. **layout** — inter-block spacing rules (template constant)
7. **substrate** — substrate overrides for substrates the page uses
8. **variant** — per-variant rules from the prototype's per-section CSS,
   with selectors rewritten to target EDS block class + variant
9. **responsive** — media queries

Step 5 (base) and step 6 (layout) are mostly TEMPLATE CONSTANTS —
the generator emits the same defensive defaults for every theme. The
content that differs per theme lives in tokens, fonts, chrome, and
variant.
