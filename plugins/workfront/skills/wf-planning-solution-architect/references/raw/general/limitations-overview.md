# Adobe Workfront Planning object limitations overview

Source: https://experienceleague.adobe.com/en/docs/workfront/using/adobe-workfront-planning/adobe-workfront-planning-general-information/limitations-overview
Last update: Wed Apr 01 2026

Adobe Workfront Planning has limits on how many objects you can create. Limits are subject to change.

| Object | Limit |
|---|---|
| Workspaces per Workfront instance | Unlimited (but recommended not to have too many) |
| Sections per workspace | 50 |
| Record types per workspace | 100 (includes record types from all sections and template-created ones) |
| Records per record type | 25,000 |
| Records per workspace | Select: 25,000 / Prime: 500,000 / Ultimate: 1,000,000 |
| Total records per WFP instance | Select: 500,000 / Prime: 2,000,000 / Ultimate: Unlimited |
| Fields per record type or taxonomy | 500 |
| Characters per Single-line text field | 1,000 |
| Characters per paragraph field | 10,000 |
| Paragraph fields per record type | 20 |
| File size for table import | 1 MB |
| File size for table import via API | 1.5 MB |
| API request rate | 200 requests per minute |
| Views per record type per user | 100 |
| CSV/Excel size for record type creation | 5 MB |
| Rows in import CSV/Excel | 25,000 |
| Columns in import CSV/Excel | 500 |
| Formula fields per record type | 20 |
| Connection fields per record type | 30 |
| Characters in formula field expression | 50,000 |
| Sharing entities per WFP object | 100 |
| Record types per hierarchy | 4 |
| Hierarchies per workspace | 5 |
| Parent records connected to one child record inside a hierarchy | 10 |
| Records connected to one record in multi-select connection (no hierarchy) | 500 |

Note: too many workspaces become hard to manage and fragment workflows.
