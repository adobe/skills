# Record types overview

Source: https://experienceleague.adobe.com/en/docs/workfront/using/adobe-workfront-planning/adobe-workfront-planning-architecture/overview-of-record-types
Last update: Thu Jan 15 2026

WFP object types are called "record types". Building blocks of workspaces.

## Key points

- Custom record types created per org's needs.
- When using a template, types organize into two sections:
  - **Operational Record Types**: strategic plans, initiatives, planned work. E.g., Campaign, Activity, Tactic, Opportunity.
  - **Taxonomies**: attributes of operational types. E.g., Region, Address, Audience.
- Sections and types can be renamed, deleted, or created.
- From-scratch workspaces let you place types in any section you create.
- On creation, only the creator and permission grantees access the type.
- Must create workspace before creating record types.

## Cross-workspace capabilities

To use record types in more than one workspace:
- **Global record types**: can be added as existing types to other workspaces.
- **Connectable record types**: can be connected to from other workspaces.

See: Configure cross-workspace capabilities for record types.
