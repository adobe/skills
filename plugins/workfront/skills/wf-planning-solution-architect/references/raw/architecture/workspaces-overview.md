# Workspaces overview

Source: https://experienceleague.adobe.com/en/docs/workfront/using/adobe-workfront-planning/adobe-workfront-planning-architecture/workspaces-overview
Last update: Fri Mar 21 2025

A workspace is a collection of record types used by an organizational unit, representing its work lifecycle and processes. Fully customizable.

## Considerations

- Create workspaces for specific organizational units to match how each unit works.
- WFP doesn't come with preconfigured workspaces. You create them.
- Create workspaces:
  - From scratch
  - Using a template (preconfigured record types and fields)
- Workspaces are frameworks for org units (team, group, department, division). They cannot have fields directly. Only record types within them have fields.
- For system admins:
  - Workspaces I'm on: created or shared with admin
  - Other workspaces: everything else
- For non-admins: only workspaces they created or are shared with them.
- Record types in a workspace should reflect the unit's lifecycle/concepts.
  - Example: campaigns, products, regions → record types Campaign, Product, Region.
- On creation, only the creator has access. Must share for collaboration.
- System admins can manage workspaces they didn't create.
- Limits apply (see limitations-overview.md).
