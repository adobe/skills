# Roll out your strategic home: a 30-day launchpad

Source: https://experienceleague.adobe.com/en/docs/workfront/using/adobe-workfront-planning/adobe-workfront-planning-best-practices/30-day-launchpad
Last update: April 1, 2026

Requires Planning Prime or higher. Aimed at admins.

## Philosophy: team-first adoption
- Avoid enterprise-wide rollouts; they cause "technology-blame".
- Decentralized, team-led approach delivering quick ROI to a single functional unit.

## "Guided autonomy" model
- Admin defines governed defaults and templates (the "lanes on the road").
- Teams choose their path within those lanes.
- Evolutionary: start with minimal global definitions for the pilot team's needs, observe patterns, iterate.

## Two pilot paths (first 15 days)
- **Parallel path**: team keeps current workflow AND models next planning cycle in WFP. Builds muscle memory without single point of failure.
- **Clean break**: after a sample-data design exercise, team moves live into WFP for next cycle.

## 30-day timeline

### Days 1-10 — Team foundation
Adopt hub-and-spoke architecture from day one:
- **Global classification workspace** (hub): system-wide structure and nomenclature.
- **Team workspace** (spoke): pilot team's lifecycle.

Design principles:
- Hub supports the spoke (focus globals on objects the pilot team cares about).
- Admin role = enabler, not guardian.
- Collaborate with pilot team to define record types and fields.
- Connect strategy to work (Planning ↔ Workfront).
- Mirror current data (use familiar inputs like Excel).
- Built for flexibility — start team-level, elevate later.

### Day 15 — First breakthrough
Target: **automated project creation**.
- Implement automation: "Ready to execute" status triggers Workfront project creation with auto-connection.
- This is the proof of value moment.

### Days 16-30 — Toward enterprise scale
- Introduce intake request forms as the data-integrity gateway.
- Observe pilot team patterns; let those shape global classification.
- Decommission legacy spreadsheets gradually.
- Train power users on the record table view for high-velocity plan management.

## Success markers
- Adoption: first 5–10 strategic records successfully linked to execution projects.
- Advocacy: pilot team starts evangelizing.
- Efficiency: time from strategic approval to project start drops.
- Trust: stakeholders use WFP timelines instead of status decks.

## Dos
- Start with an excited pilot team.
- Define lanes, not walls.
- Prioritize psychological safety (legacy tools stay until comfort exists).
- Celebrate quick wins.
- Establish a champion per team.

## Don'ts
- Don't mandate large-scale changes day one.
- Don't ignore the "why" — explain how the strategic thread makes work more visible.
- Don't wait for taxonomy perfection.
